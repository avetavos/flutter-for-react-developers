ระบบ structural type ของ TypeScript ช่วยให้คุณนิยาม `interface` และใช้ผลลัพธ์จาก `JSON.parse` เป็น type นั้นได้ทันที — compiler เชื่อ shape ที่คุณประกาศ Dart ใช้ nominal typing และไม่มี runtime structural casting คุณต้องเขียน `fromJson` factory constructor ที่อ่านแต่ละ field จาก decoded map โดยตรง โค้ดมากกว่า แต่ explicit มากกว่าด้วย: ถ้า field ขาดหายหรือ type ไม่ตรง คุณจะรู้ทันที ไม่ใช่ตอน call site

## เปรียบเทียบแบบ side-by-side

export const jsonReact = `// TypeScript — interface + JSON.parse
interface User {
  id: number;
  name: string;
  email: string;
}

// TypeScript trusts the shape (no runtime check)
const user = JSON.parse(raw) as User;
console.log(user.name);

// With Zod for runtime validation
import { z } from 'zod';

const UserSchema = z.object({
  id: z.number(),
  name: z.string(),
  email: z.string().email(),
});

const safeUser = UserSchema.parse(JSON.parse(raw));`;

export const jsonFlutter = `// Dart — class with fromJson / toJson
import 'dart:convert';

class User {
  final int id;
  final String name;
  final String email;

  const User({
    required this.id,
    required this.name,
    required this.email,
  });

  // Explicit field mapping — no structural magic
  factory User.fromJson(Map<String, dynamic> json) {
    return User(
      id: json['id'] as int,
      name: json['name'] as String,
      email: json['email'] as String,
    );
  }

  Map<String, dynamic> toJson() => {
    'id': id,
    'name': name,
    'email': email,
  };
}

// Usage
final map = jsonDecode(raw) as Map<String, dynamic>;
final user = User.fromJson(map);
print(user.name);`;

<Diff title="Structural typing vs explicit fromJson">

- **ไม่มี structural casting.** TypeScript ยอมรับ `JSON.parse(raw) as User` เพราะ structurally ตรงกับ interface ตอน compile time Dart ปฏิเสธ pattern นี้ — คุณต้องเรียก `User.fromJson(map)` เพื่อแปลง untyped `Map<String, dynamic>` เป็น `User` instance จริง
- **`factory` constructors.** keyword `factory` ใน Dart ทำเครื่องหมาย constructor ที่อาจ return cached instance หรือ delegate ไปยัง constructor อื่น — เป็นที่อยู่ตามธรรมเนียมของ `fromJson` เพราะสามารถมี logic ใด ๆ ก่อน call constructor จริง
- **`as` cast คือ runtime check.** การเขียน `json['id'] as int` ใน Dart จะ throw `TypeError` ตอน runtime ถ้าค่าไม่ใช่ `int` นี่คือ feature ไม่ใช่ bug: คุณได้ error ที่ชัดเจนตั้งแต่ต้น ไม่ใช่ `undefined` แบบเงียบ ๆ ภายหลัง
- **`toJson` ก็ต้องเขียนเองเช่นกัน.** ไม่มี built-in serialization method `toJson` คืน `Map<String, dynamic>` ที่ `jsonEncode` (จาก `dart:convert`) สามารถแปลงเป็น string ได้
- **Codegen สำหรับ model ขนาดใหญ่.** สำหรับ production apps พิจารณาใช้ `json_serializable` (generate `fromJson`/`toJson`) หรือ `freezed` (เพิ่ม immutability, `copyWith`, และ union types) ทั้งคู่ช่วยขจัด boilerplate ที่ต้องเขียนมือ เหมือนที่ Zod schema ขจัด validation code ที่ซ้ำซ้อน

</Diff>

import 'package:flutter/material.dart';

class User {
