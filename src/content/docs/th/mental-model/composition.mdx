Composition model ของ React — component เล็กๆ ที่รับ `children`, higher-order components ที่ wrap behavior, และ render props — map ได้อย่างลงตัวกับ widget model ของ Flutter ใน Flutter คุณ compose behavior โดยการ wrap widget: `Padding(child: ...)`, `GestureDetector(child: ...)`, `Opacity(child: ...)` ไม่มี CSS class inheritance หรือ widget subclassing สำหรับการ customize รูปลักษณ์ ทุกอย่างคือ composition

## การ Wrap Behavior

export const reactComposition = `// React — children prop
function Card({ children, elevated }) {
  return (
    <div
      style={{
        padding: 16,
        borderRadius: 12,
        boxShadow: elevated ? '0 4px 12px rgba(0,0,0,0.1)' : 'none',
      }}
    >
      {children}
    </div>
  );
}

// Usage
<Card elevated>
  <h3>Hello</h3>
  <p>Composed content</p>
</Card>`;

export const flutterComposition = `// Flutter — wrapper widget with child
class AppCard extends StatelessWidget {
  final Widget child;
  final bool elevated;
  const AppCard({super.key, required this.child, this.elevated = false});

  @override
  Widget build(BuildContext context) {
    return Container(
      padding: const EdgeInsets.all(16),
      decoration: BoxDecoration(
        color: Colors.white,
        borderRadius: BorderRadius.circular(12),
        boxShadow: elevated
            ? [BoxShadow(
                color: Colors.black.withOpacity(0.1),
                blurRadius: 12,
                offset: const Offset(0, 4))]
            : [],
      ),
      child: child,
    );
  }
}

// Usage
AppCard(
  elevated: true,
  child: Column(
    children: [
      Text('Hello', style: TextStyle(fontSize: 18, fontWeight: FontWeight.bold)),
      Text('Composed content'),
    ],
  ),
)`;

<Diff title="React children / HOCs → Flutter composition">

- **`{children}` ใน JSX** → parameter `child: Widget` (single child) หรือ `children: List<Widget>` (multiple children)
- **HOC wrapping behavior** → Wrap ด้วย widget: `GestureDetector(onTap: ..., child: myWidget)` — แต่ละ wrapper เพิ่ม behavior เดียว
- **Render props** → ส่ง `Widget Function(...)` เป็น constructor parameter — parent เรียกมันแล้วใช้ widget ที่ return มา
- **`React.Children.map`** → `children` เป็น `List<Widget>` ธรรมดา — เรียก `.map()` ตรงๆ ได้เลย
- **ไม่มี inheritance สำหรับ visual customization** — compose เสมอ, อย่า extend — คุณไม่สามารถ style widget โดยการ subclass มัน

</Diff>
